### Simulation Rules (implement exactly):
Maintain two 2D arrays: `grid` (walls/floor/start/goal) and `scent` (non-negative floats, same dimensions).

1. Agents:
   - Run exactly 40 agents per simulation.
   - All agents spawn on the Start tile when the maze loads or resets.
   - Each tick, each agent moves exactly one cell to an adjacent orthogonal neighbor (up/down/left/right). No diagonal moves. Cannot enter Wall cells.
   - If an agent reaches the Goal tile, remove it from the active set (it "arrived") and increment an arrived counter.

2. Scent deposit:
   - After each agent moves, add 0.8 scent only to the cell it **left** (trail behind). Do not deposit on the cell the agent enters.
   - Cap scent on the Start tile at 2.0 (clamp after deposit) so spawn point does not become an immovable attractor.

3. Scent decay:
   - At the start of every tick, multiply every cell's scent value by 0.90 (10% decay per tick). Apply decay before agent movement and deposit.

4. Movement choice (the core algorithm):
   Each agent stores the grid coordinates of the cell it occupied on the previous tick (`previousX`, `previousY`; `null` on first move after spawn).

   a) Build candidates: all valid orthogonal neighbors (in bounds, not Wall).

   b) Anti-backtracking: if there are 2 or more candidates, remove the `previous` cell from candidates. If that leaves zero candidates, allow all valid neighbors (dead-end exception).

   c) Exploration (15%): with probability 0.15, pick uniformly at random among candidates and move there — skip steps (d)–(e) for this agent this tick.

   d) Gradient weights: for each candidate, compute
      `weight = max(0, scent[candidate] - scent[current]) + 0.05`
      This follows the scent **gradient**, not absolute scent — agents move toward stronger signal ahead, not back and forth between two hot neighbors.

   e) Weighted choice: pick a candidate by weighted random using the weights from (d). If every weight is identical, break ties uniformly at random.

5. Anti-oscillation (required):
   - Agents must spread through the maze over time; bouncing between two adjacent tiles is a bug.
   - The combination of gradient weights, anti-backtracking, one-sided deposit, and 15% exploration must prevent 2-cell loops. Do not implement movement that only compares raw `scent[neighbor]` values without subtracting `scent[current]`.

6. Loop:
   - Drive the simulation with `requestAnimationFrame` (or a fixed-interval timer at ~30 fps).
   - Simulation should run continuously until paused or reset; pausing is optional but not required.

### Critical Implementation Rules:

- Each agent must update `previousX`/`previousY` after every move.
- When 40 agents share the Start tile, they will disperse via random exploration and anti-backtracking — do not special-case Start beyond the scent cap.
